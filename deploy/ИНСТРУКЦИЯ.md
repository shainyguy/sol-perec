## Шаг 1 — Подготовка VPS

Подключитесь по SSH:
```bash
ssh root@ВАШ_IP
```

Обновите систему и установите зависимости:
```bash
apt update && apt upgrade -y
apt install -y nodejs npm postgresql nginx git

# Проверьте версию Node (нужна 18+)
node --version

# Если версия старая — установите через nvm:
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
source ~/.bashrc
nvm install 20
nvm use 20

# Установите PM2 глобально
npm install -g pm2
```

---

## Шаг 2 — Настройка PostgreSQL

```bash
# Войдите в PostgreSQL
sudo -u postgres psql
```

В консоли PostgreSQL выполните:
```sql
CREATE USER sol_perec_user WITH PASSWORD 'ПРИДУМАЙТЕ_СЛОЖНЫЙ_ПАРОЛЬ';
CREATE DATABASE sol_perec OWNER sol_perec_user;
GRANT ALL PRIVILEGES ON DATABASE sol_perec TO sol_perec_user;
\q
```

Создайте таблицы:
```bash
psql -U sol_perec_user -d sol_perec -f /var/www/sol-perec/deploy/init.sql
```

## Шаг 3 — Загрузка файлов сайта

### Вариант А — через Git (если есть репозиторий):
```bash
mkdir -p /var/www/sol-perec
cd /var/www/sol-perec
git clone ВАШ_РЕПОЗИТОРИЙ .
```

### Вариант Б — через FTP/SCP (загрузить архив):
```bash
# На локальном компьютере — собрать и упаковать:
npm run build
tar -czf sol-perec.tar.gz dist/ server.js package.json package-lock.json public/ deploy/

# Загрузить на сервер:
scp sol-perec.tar.gz root@ВАШ_IP:/var/www/

# На сервере — распаковать:
mkdir -p /var/www/sol-perec
cd /var/www/sol-perec
tar -xzf /var/www/sol-perec.tar.gz
```

---

## Шаг 4 — Настройка .env

```bash
cd /var/www/sol-perec
cp deploy/.env.example .env
nano .env  # или vim .env
```

Заполните параметры:
```env
DB_HOST=localhost
DB_PORT=5432
DB_NAME=sol_perec
DB_USER=sol_perec_user
DB_PASSWORD=ВАШ_ПАРОЛЬ_ИЗ_ШАГА_2
DB_SSL=false
PORT=3000
NODE_ENV=production
ADMIN_PASSWORD=придумайте_пароль_для_админки
```

---

## Шаг 5 — Установка зависимостей и запуск

```bash
cd /var/www/sol-perec
npm install --production

# Создать папку для логов
mkdir -p logs

# Запустить через PM2
pm2 start deploy/ecosystem.config.cjs

# Проверить статус
pm2 status
pm2 logs sol-perec

# Добавить в автозапуск
pm2 startup
pm2 save
```

Проверьте что сервер работает:
```bash
curl http://localhost:3000
```

## Шаг 6 — Настройка Nginx

```bash
# Скопировать конфиг
cp /var/www/sol-perec/deploy/nginx.conf /etc/nginx/sites-available/sol-perec

# Отредактировать — заменить ВАШ_ДОМЕН.ru на реальный домен
nano /etc/nginx/sites-available/sol-perec

# Активировать
ln -s /etc/nginx/sites-available/sol-perec /etc/nginx/sites-enabled/

# Удалить дефолтный конфиг
rm -f /etc/nginx/sites-enabled/default

# Проверить и перезапустить
nginx -t
systemctl reload nginx
```

## Перенос данных из Supabase

Если нужно перенести данные из текущей Supabase в локальную PostgreSQL:

```bash
# Экспорт из Supabase (выполнить локально)
pg_dump "postgresql://postgres:ПАРОЛЬ@db.ПРОЕКТ.supabase.co:5432/postgres" \
  --table=menu_items --table=reviews --table=gallery --table=promotions \
  --data-only --no-owner -f supabase_data.sql

# Импорт на сервер
psql -U sol_perec_user -d sol_perec -f supabase_data.sql
```

## Полезные команды

```bash
# Просмотр логов
pm2 logs sol-perec --lines 100

# Перезапуск после обновления кода
pm2 restart sol-perec

# Статус
pm2 status

# Обновление сайта
cd /var/www/sol-perec
git pull  # если через git
npm run build
pm2 restart sol-perec

# Бэкап БД
pg_dump -U sol_perec_user sol_perec > backup_$(date +%Y%m%d).sql

# Автобэкап (добавить в crontab -e)
# 0 3 * * * pg_dump -U sol_perec_user sol_perec > /var/backups/sol_perec_$(date +\%Y\%m\%d).sql
```

## ❓ Частые проблемы

**Сайт не открывается:**
```bash
pm2 status          # проверить запущен ли процесс
pm2 logs sol-perec  # смотреть ошибки
nginx -t            # проверить конфиг nginx
systemctl status nginx
```

**Ошибка подключения к БД:**
```bash
psql -U sol_perec_user -d sol_perec  # проверить подключение
cat .env                              # проверить параметры
```

**Порт занят:**
```bash
lsof -i :3000       # кто занимает порт
kill -9 PID         # убить процесс
pm2 start deploy/ecosystem.config.cjs
```
